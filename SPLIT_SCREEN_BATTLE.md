# 分屏对战功能说明

## 概述

本次更新实现了双人游戏的分屏对战界面。在玩家2登录成功后，系统会显示一个分屏界面，分别展示玩家1和玩家2的信息，包括：
- 玩家头像和昵称
- 当前分数
- 准备状态

## 功能流程

### 1. 完整流程

1. **玩家1登录**：在登录场景输入昵称
2. **选择游戏**：在主菜单选择一个游戏（颜色游戏、形状游戏或数字游戏）
3. **选择对战模式**：点击"双人对战"按钮
4. **玩家2登录**：
   - 显示玩家1的登录信息
   - 玩家2输入昵称和密码
   - 点击"开始对战"按钮
5. **分屏界面**：
   - 显示两个玩家的信息面板
   - 左侧显示玩家1（红色主题）
   - 右侧显示玩家2（蓝色主题）
   - 中间有分割线
   - 底部有"开始对战"按钮
6. **开始游戏**：点击"开始对战"进入实际游戏场景

### 信息面板

每个玩家面板包含：

1. **标题**：🎮 玩家1 / 玩家2

## 关键更改

### 1. SplitScreenBattleScene（新增）

**位置**：`src/scenes/split-screen-battle.scene.ts`

**主要功能**：
- 创建分屏布局（左中右三个区域）
- 显示两个玩家的信息面板
- 实现玩家1和玩家2的视觉区分
- 提供"开始对战"按钮
- 支持返回模式选择界面

**关键方法**：
- `createSplitScreenLayout()`：创建分屏布局和分割线
- `createPlayerInfoPanels()`：创建两个玩家的信息面板
- `createPlayer1Panel()`：创建玩家1的面板（红色主题）
- `createPlayer2Panel()`：创建玩家2的面板（蓝色主题）
- `createStartButton()`：创建"开始对战"按钮
- `createBackButton()`：创建返回按钮
- `startBattleGame()`：启动实际游戏场景
- `updateScore()`：更新分数显示

### 2. Player2LoginScene（修改）

**位置**：`src/scenes/player2-login.scene.ts`

**更改**：
- 修改 `handleLogin()` 方法
- 玩家2登录成功后，调用 `startSplitScreen()` 而不是 `startGame()`
- 保留对战信息设置逻辑

### 3. main.ts（修改）

**更改**：
- 导入 `SplitScreenBattleScene`
- 在场景注册表中注册该场景
- 优先级设置为92（在玩家2登录之后）

## 全局变量

### `__BATTLE_INFO__`

对战信息结构：

```typescript
{
  gameId: 'color-game',  // 游戏ID
  players: [              // 玩家信息数组
    { id: 'user1_id', username: '小明' },
    { id: 'user2_id', username: '小红' }
  ]
}
```

### `__GAME_ID__`

当前选择的游戏ID：
- `'color-game'`：颜色游戏
- `'shape-game'`：形状游戏
- `'demo-game'`：数字游戏

## UI组件详情

### 玩家面板

**尺寸**：
- 宽度：300 * scaleFactor
- 高度：400 * scaleFactor

**样式**：
- 背景：半透明彩色（红色或蓝色）
- 边框：同色系，透明度0.5
- 圆角：20px

**字体大小**：
- 标题：28px * scaleFactor
- 昵称：24px * scaleFactor
- 分数标签：18px * scaleFactor
- 分数值：48px * scaleFactor
- 状态：16px * scaleFactor

### 开始按钮

**尺寸**：
- 宽度：200 * scaleFactor
- 高度：60 * scaleFactor

**样式**：
- 渐变背景：绿色（#4CAF50 到 #45A049）
- 文字：白色，24px
- 圆角：30px
- 悬停时放大1.05倍

### 分割线

**样式**：
- 线条：4px宽度
- 颜色：浅灰色 (#E0E0E0)
- 位置：屏幕中央垂直线

## 交互行为

### 按钮交互

- **鼠标悬停**：按钮放大1.05倍
- **鼠标移出**：按钮恢复原大小
- **点击**：触发对应功能

### 返回按钮

- 点击返回模式选择场景（`BattleSelectScene`）

### 开始按钮

- 点击启动实际游戏场景（根据 `__GAME_ID__` 决定）
- 设置 `__GAME_BATTLE_MODE__ = 'battle'`

## 响应式设计

场景使用 `DeviceUtil.getScaleFactor()` 进行缩放，确保在不同屏幕尺寸下都能正常显示。

**适配规则**：
- 字体大小乘以 scaleFactor
- 组件尺寸乘以 scaleFactor
- 位置根据屏幕百分比计算

## 注意事项

1. **当前实现**：分屏界面只是游戏前的准备界面，游戏内仍是轮流制
2. **真实分屏**：如需真正的分屏游戏（两个玩家同时玩），需要使用 `SplitScreenGameManager`
3. **屏幕尺寸**：建议使用横屏模式，两个玩家面板并排显示更清晰
4. **性能**：当前实现使用单个Phaser场景，性能开销较小
