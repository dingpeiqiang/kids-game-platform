# 玩家2登录功能说明

## 概述

本次更新实现了双人游戏模式中的玩家2登录功能。在选择双人对战模式后，系统会要求玩家2输入用户名和密码进行认证，认证通过后才开始游戏。

## 功能流程

### 1. 进入对战模式

1. 玩家1首先登录游戏
2. 选择游戏（颜色游戏、形状游戏或数字游戏）
3. 在模式选择界面选择"双人对战"模式

### 2. 玩家2登录

1. 系统显示玩家2登录界面
2. 界面显示玩家1的登录信息（用户名和头像）
3. 玩家2需要输入：
   - 昵称（3-10个字符）
   - 密码（至少4个字符）
4. 系统会验证：
   - 昵称和密码是否符合长度要求
   - 昵称不能与玩家1相同
   - 昵称是否已被注册

### 3. 认证流程

- **新用户注册**：如果昵称未被注册，系统会自动注册并登录玩家2
- **现有用户登录**：如果昵称已被注册，需要输入正确的密码
- **验证失败**：如果昵称或密码不正确，显示错误提示

### 4. 开始游戏

- 玩家2登录成功后，系统会设置对战信息
- 两个玩家的信息存储到全局变量 `__BATTLE_INFO__` 中
- 自动启动对应游戏的场景

## 关键更改

### 1. Player2LoginScene (新增场景)

**位置**：`src/scenes/player2-login.scene.ts`

**主要功能**：
- 显示玩家1的登录信息
- 提供玩家2的昵称和密码输入框
- 实现输入框聚焦和光标动画
- 验证玩家2的登录信息
- 注册并登录玩家2
- 设置对战信息并启动游戏

**UI组件**：
- 标题区域
- 玩家1信息卡片
- 昵称输入框
- 密码输入框（密码以圆点显示）
- "开始对战"按钮
- 返回按钮

### 2. BattleSelectScene (修改)

**位置**：`src/core/battle-select.scene.ts`

**更改**：
- 修改 `selectMode()` 方法
- 当选择"双人对战"模式时，跳转到 `Player2LoginScene` 而不是 `BattleRequestScene`
- 存储游戏ID到全局变量 `__GAME_ID__`

### 3. UserService (已有功能)

**位置**：`src/services/user.service.ts`

**使用功能**：
- `register(username, password)`：注册新用户
- `getCurrentUser()`：获取当前登录用户（玩家1）
- 已支持用户注册和密码存储功能

## 全局变量

### `__GAME_ID__`

存储当前选择的游戏ID：
- `'color-game'`：颜色游戏
- `'shape-game'`：形状游戏
- `'demo-game'`：数字游戏

### `__BATTLE_INFO__`

存储对战信息：

```typescript
{
  gameId: 'color-game',  // 游戏ID
  players: [              // 玩家信息数组
    { id: 'user1_id', username: '玩家1昵称' },
    { id: 'user2_id', username: '玩家2昵称' }
  ]
}
```

### `__PLAYER2_INFO__`

存储玩家2的详细信息：

```typescript
{
  id: 'user2_id',
  username: '玩家2昵称',
  avatar: '🐱',
  status: 'online',
  points: 100
}
```

## 使用示例

### 正常流程

1. 玩家1登录：昵称"小明"
2. 选择颜色游戏 → 双人对战
3. 进入玩家2登录界面
4. 玩家2输入：昵称"小红"，密码"123456"
5. 系统注册玩家2并登录
6. 自动启动颜色游戏，两个玩家信息已设置

### 错误处理

**昵称太短**：
- 显示："昵称至少需要3个字符哦～"

**密码太短**：
- 显示："密码至少需要4个字符～"

**昵称与玩家1相同**：
- 显示："不能与玩家1使用相同的昵称～"

**昵称已注册且密码错误**：
- 显示："用户名或密码错误"（需要在 UserService 中实现）

## 键盘快捷键

- `Tab`：在昵称和密码输入框之间切换
- `Enter`：提交登录
- `Backspace`：删除字符
- `Esc`：返回模式选择界面

## 交互说明

### 输入框

- 点击输入框聚焦
- 输入时显示光标
- 密码输入框显示圆点（●）而非明文

### 按钮

- 鼠标悬停时放大（1.05倍）
- 点击时触发对应操作
- 登录过程中禁用按钮，防止重复提交

## 技术细节

### 输入处理

- 使用 Phaser 的键盘事件监听
- 支持中文、字母、数字输入
- 过滤特殊字符

### 光标动画

- 聚焦时显示光标
- 光标每500ms闪烁一次
- 取消聚焦时光标消失

### 密码显示

- 实际密码存储在 `this.password` 中
- UI显示使用圆点（●）代替
- 圆点数量与密码长度相同

1. 当前使用 `localStorage` 存储用户数据
2. 密码使用 Base64 简单编码（仅用于演示，生产环境应使用更安全的加密方式）
3. 单机双人对战模式，两个玩家在同一设备上操作
4. 如果需要远程对战，需要实现 WebSocket 或后端服务器
